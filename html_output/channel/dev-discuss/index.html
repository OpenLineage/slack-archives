<p><strong>Airflow Integration</strong></p> <p>:::note</p> <p>If you're looking to schedule DataHub ingestion using Airflow, see the guide on &lt;../../metadata-ingestion/schedule_docs/airflow.md|scheduling ingestion with Airflow>.</p> <p>:::</p> <p>The DataHub Airflow plugin supports:</p> <p>• Automatic column-level lineage extraction from various operators e.g. <code>SqlOperator</code>s (including <code>MySqlOperator</code>, <code>PostgresOperator</code>, <code>SnowflakeOperator</code>, and more), <code>S3FileTransformOperator</code>, and a few others. • Airflow DAG and tasks, including properties, ownership, and tags. • Task run information, including task successes and failures. • Manual lineage annotations using <code>inlets</code> and <code>outlets</code> on Airflow operators.</p> <p>There's two actively supported implementations of the plugin, with different Airflow version support.</p> <p>If you're using Airflow older than 2.1, it's possible to use the v1 plugin with older versions of <code>acryl-datahub-airflow-plugin</code>. See the <a href="https://github.com/datahub-project/datahub/blob/2b0811b9875d7d7ea11fb01d0157a21fdd67f020/docs/lineage/airflow.md#compatibility">compatibility section</a> for more details.</p> <p><strong>DataHub Plugin v2</strong> <strong>Installation</strong></p> <p>The v2 plugin requires Airflow 2.3+ and Python 3.8+. If you don't meet these requirements, use the v1 plugin instead.</p> <pre><code>pip install 'acryl-datahub-airflow-plugin[plugin-v2]' </code></pre> <p><strong>Configuration</strong></p> <p>Set up a DataHub connection in Airflow, either via command line or the Airflow UI.</p> <p><strong>Command Line</strong></p> <pre><code>airflow connections add --conn-type 'datahub-rest' 'datahub_rest_default' --conn-host '&lt;http://datahub-gms:8080&gt;' --conn-password '&lt;optional datahub auth token&gt;' </code></pre> <p><strong>Airflow UI</strong></p> <p>On the Airflow UI, go to Admin -> Connections and click the "+" symbol to create a new connection. Select "DataHub REST Server" from the dropdown for "Connection Type" and enter the appropriate values.</p> <p><a href="https://raw.githubusercontent.com/datahub-project/static-assets/main/imgs/airflow/plugin_connection_setup.png">https://raw.githubusercontent.com/datahub-project/static-assets/main/imgs/airflow/plugin<em>connection</em>setup.png</a></p> <p><strong>Optional Configurations</strong></p> <p>No additional configuration is required to use the plugin. However, there are some optional configuration parameters that can be set in the <code>airflow.cfg</code> file.</p> <pre><code>[datahub] # Optional - additional config here. enabled = True # default </code></pre> <p><strong>Automatic lineage extraction</strong></p> <p>To automatically extract lineage information, the v2 plugin builds on top of Airflow's built-in <a href="https://openlineage.io/docs/integrations/airflow/default-extractors">OpenLineage extractors</a>.</p> <p>The SQL-related extractors have been updated to use DataHub's SQL parser, which is more robust than the built-in one and uses DataHub's metadata information to generate column-level lineage. We discussed the DataHub SQL parser, including why schema-aware parsing works better and how it performs on benchmarks, during the <https://youtu.be/1QVcUmRQK5E?si=U27zygR7Gi_KdkzE&amp;t=2309|June 2023 community town hall>.</p> <p><strong>DataHub Plugin v1</strong> <strong>Installation</strong></p> <p>The v1 plugin requires Airflow 2.1+ and Python 3.8+. If you're on older versions, it's still possible to use an older version of the plugin. See the <a href="https://github.com/datahub-project/datahub/blob/2b0811b9875d7d7ea11fb01d0157a21fdd67f020/docs/lineage/airflow.md#compatibility">compatibility section</a> for more details.</p> <p>If you're using Airflow 2.3+, we recommend using the v2 plugin instead. If you need to use the v1 plugin with Airflow 2.3+, you must also set the environment variable <code>DATAHUB_AIRFLOW_PLUGIN_USE_V1_PLUGIN=true</code>.</p> <pre><code>pip install 'acryl-datahub-airflow-plugin[plugin-v1]' # The DataHub rest connection type is included by default. # To use the DataHub Kafka connection type, install the plugin with the kafka extras. pip install 'acryl-datahub-airflow-plugin[plugin-v1,datahub-kafka]' </code></pre> <p><strong>Configuration</strong> <strong>Disable lazy plugin loading</strong></p> <pre><code>[core] lazy_load_plugins = False </code></pre> <p>On MWAA you should add this config to your <a href="https://docs.aws.amazon.com/mwaa/latest/userguide/configuring-env-variables.html#configuring-2.0-airflow-override">Apache Airflow configuration options</a>.</p> <p><strong>Setup a DataHub connection</strong></p> <p>You must configure an Airflow connection for Datahub. We support both a Datahub REST and a Kafka-based connections, but you only need one.</p> <pre><code># For REST-based: airflow connections add --conn-type 'datahub_rest' 'datahub_rest_default' --conn-host '&lt;http://datahub-gms:8080&gt;' --conn-password '&lt;optional datahub auth token&gt;' # For Kafka-based (standard Kafka sink config can be passed via extras): airflow connections add --conn-type 'datahub_kafka' 'datahub_kafka_default' --conn-host 'broker:9092' --conn-extra '{}' </code></pre> <p><strong>Configure the plugin</strong></p> <p>If your config doesn't align with the default values, you can configure the plugin in your <code>airflow.cfg</code> file.</p> <pre><code>[datahub] enabled = true conn_id = datahub_rest_default # or datahub_kafka_default # etc. </code></pre> <p><strong>Validate that the plugin is working</strong></p> <ol><li>Go and check in Airflow at Admin -> Plugins menu if you can see the DataHub plugin</li><li>Run an Airflow DAG. In the task logs, you should see Datahub related log messages like:</li> </ol> <pre><code>Emitting DataHub ... </code></pre> <p><strong>Manual Lineage Annotation</strong> <strong>Using <code>inlets</code> and <code>outlets</code></strong></p> <p>You can manually annotate lineage by setting <code>inlets</code> and <code>outlets</code> on your Airflow operators. This is useful if you're using an operator that doesn't support automatic lineage extraction, or if you want to override the automatic lineage extraction.</p> <p>We have a few code samples that demonstrate how to use <code>inlets</code> and <code>outlets</code>:</p> <p>• &lt;../../metadata-ingestion-modules/airflow-plugin/src/datahub<em>airflow</em>plugin/example<em>dags/lineage</em>backend<em>demo.py|<code>lineage_backend_demo.py</code>&gt;
